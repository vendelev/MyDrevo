# Примеры заполненных файлов

Эта папка содержит примеры реальных файлов, которые создаются AI-агентами на разных этапах разработки.

## Список примеров

### 1. EpicSummary-example.md

**Создается на**: Шаг 0 (агент `epic-writer`)

**Содержит**:

- Описание бизнес-требований для эпика
- Разбивку на этапы с приоритизацией
- Критерии успеха и риски

**Использование**: Смотрите этот файл, чтобы понять, как должен выглядеть результат создания эпика.

---

### 2. Spec-example.md

**Создается на**: Шаг 1 (агент `feature-writer`)

**Содержит**:

- Детальное описание функциональных требований
- Модель предметной области
- Сценарии использования
- Критерии приемки

**Использование**: Пример спецификации для одного этапа эпика или для отдельной фичи.

---

### 3. TaskSummary-example.md

**Создается на**: Шаг 2 (агент `summary-plan-writer`)

**Содержит**:

- Архитектурное решение
- Разбивку на технические задачи
- Диаграммы потоков данных
- Конфигурацию и риски

**Использование**: Пример сводного технического плана разработки.

---

## Как использовать примеры

### При создании эпика

1. Откройте `EpicSummary-example.md`
2. Посмотрите структуру разделов
3. Обратите внимание на:
   - Как сформулированы бизнес-цели
   - Как приоритизированы этапы
   - Как описаны критерии успеха

### При создании спецификации

1. Откройте `Spec-example.md`
2. Изучите:
   - Формулировку функциональных требований
   - Структуру модели предметной области
   - Детальность сценариев использования

### При создании технического плана

1. Откройте `TaskSummary-example.md`
2. Обратите внимание на:
   - Архитектурные решения
   - Разбивку на атомарные задачи
   - Зависимости между задачами

---

## Кастомизация под свой проект

Эти примеры основаны на гипотетическом проекте. Для вашего проекта:

1. Адаптируйте терминологию под свой домен
2. Скорректируйте структуру под свои требования
3. Добавьте специфичные разделы (например, безопасность, интеграции)
