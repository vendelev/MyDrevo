## Workflow для крупной задачи (через эпик)

### □ Шаг 0: Создание эпика

```bash
/ra-create-epic <номер_эпика> <описание_бизнес_потребности>
```

**Результат**:

- [ ] Создана папка `/Doc/Backlog/{YYYY}/{EPIC_FOLDER}/`
- [ ] Файл `EpicSummary.md` содержит бизнес-требования
- [ ] Файлы `Stage1.md`, `Stage2.md`, ... созданы для каждого этапа
- [ ] Ревью пройдено
- [ ] Коммит сделан

---

### □ Шаг 1: Создание спецификации (для каждого этапа)

```bash
/ra-create-feature <номер_задачи> <путь_к_EpicSummary.md> <номер_этапа>
```

**Результат**:

- [ ] Создана папка `/Doc/FeatureList/{YYYY}/{MM}/{FEATURE_FOLDER}/`
- [ ] Файл `Spec.md` содержит функциональные требования
- [ ] Модель предметной области описана
- [ ] Сценарии использования приведены
- [ ] Ревью пройдено
- [ ] Коммит сделан

---

### □ Шаг 2: Создание сводного плана

**Внимание**: Создавайте непосредственно перед разработкой!

```bash
/ra-create-summary-plan @/Doc/FeatureList/{YYYY}/{MM}/{FEATURE_FOLDER}/Spec.md
```

**Результат**:

- [ ] Файл `TaskSummary.md` создан
- [ ] Архитектурное решение описано
- [ ] Задачи последовательны и атомарны
- [ ] Ревью пройдено
- [ ] Коммит сделан

---

### □ Шаг 3: План разработки (для каждой задачи)

```bash
/ra-create-dev-plan <номер_задачи> @/Doc/FeatureList/{YYYY}/{MM}/{FEATURE_FOLDER}
```

**Результат**:

- [ ] Файл `TaskList/TaskX_TaskForDev.md` создан
- [ ] Последовательность реализации понятна
- [ ] Все файлы и пути указаны
- [ ] Ревью пройдено
- [ ] Коммит сделан

---

### □ Шаг 4: План тестирования (для каждой задачи)

```bash
/ra-create-test-plan <номер_задачи> @/Doc/FeatureList/{YYYY}/{MM}/{FEATURE_FOLDER}
```

**Результат**:

- [ ] Файл `TaskList/TaskX_TaskForTest.md` создан
- [ ] Сценарии тестирования описаны
- [ ] Формат: Дано → Проверяемое → Ожидаемый результат
- [ ] Ревью пройдено
- [ ] Коммит сделан

---

### □ Шаг 5: Разработка кода (для каждой задачи)

```bash
/ra-php-implementation <номер_задачи> @/Doc/FeatureList/{YYYY}/{MM}/{FEATURE_FOLDER}/TaskSummary.md
```

**Результат**:

- [ ] Код создан согласно плану
- [ ] PHPStan не выдает ошибок
- [ ] PHPCS не выдает ошибок
- [ ] Rector применен
- [ ] Ревью пройдено
- [ ] Коммит сделан

---

### □ Шаг 6: Написание тестов (для каждой задачи)

```bash
/ra-php-test-implementation <номер_задачи> @/Doc/FeatureList/{YYYY}/{MM}/{FEATURE_FOLDER}/TaskSummary.md
```

**Результат**:

- [ ] Тесты созданы согласно плану
- [ ] PHPUnit: все тесты проходят
- [ ] Покрытие кода ≥ 75%
- [ ] PHPStan не выдает ошибок
- [ ] PHPCS не выдает ошибок
- [ ] Ревью пройдено
- [ ] Коммит сделан

---

### □ Повторить для остальных задач

- [ ] Шаги 3-8 повторены для всех задач из `TaskSummary.md`

---

- [ ] Коммит сделан

## Типичные ошибки

### ❌ Неправильно

- Создавать технические планы для всех задач сразу (до начала разработки)
- Пропускать ревью
- Игнорировать ошибки PHPStan/PHPCS
- Создавать документацию до завершения кода

### ✅ Правильно

- Создавать технический план непосредственно перед разработкой
- Проводить ревью после каждого шага
- Исправлять все ошибки статических анализаторов
- Создавать документацию после реализации и тестирования

## Оценка прогресса

### Для одной задачи

- Шаг 3: План разработки ≈ 5 минут
- Шаг 4: План тестирования ≈ 5 минут
- Шаг 5: Разработка кода ≈ 15-30 минут
- Шаг 6: Написание тестов ≈ 15-30 минут

**Итого**: ~40-70 минут на задачу

### Для этапа с 3 задачами

- Шаги 1-2: Спецификация + сводный план ≈ 20 минут
- Шаги 3-6 для 3 задач ≈ 2-3.5 часа
- Шаги 7-8: Документация ≈ 30 минут

**Итого**: ~3-4.5 часа на этап
