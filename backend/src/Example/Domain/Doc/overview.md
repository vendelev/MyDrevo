# Модуль Example

## Назначение

Модуль **Example** реализует эталонную реализацию модуля согласно принципам Clean Architecture и CQRS. 
Он демонстрирует основные паттерны и структуру для создания новых модулей в проекте.

## Архитектурная роль

- принимает HTTP-запросы через контроллеры для CRUD операций и HTML отображения;
- выполняет бизнес-логику через UseCase, команды и запросы CQRS;
- взаимодействует с таблицами `examples` и `example_outbox` в SQLite;
- выполняет математические вычисления и генерирует отчеты;
- логирует в внешнюю систему Clickhouse.

## Компоненты

### example.web

Web-контур модуля: контроллеры, middleware и Blade-шаблоны для API создания примеров и HTML отображения данных примера.

### example.crud.service

UseCase и CQRS-команды/запросы для создания и получения примеров, включая вставку в outbox и генерацию событий.

### example.calc.service

UseCase, сервис вычислений и responder для математических операций с генерацией Markdown отчетов.

### db.examples

Таблица examples, хранящая сущности примеров с полями name, comment, status, user_id, created_at.

### db.example_outbox

Таблица example_outbox для асинхронной обработки данных в формате JSON.

### external.clickhouse

Внешний адаптер для логгирования сообщений в систему Clickhouse с API ключом.
