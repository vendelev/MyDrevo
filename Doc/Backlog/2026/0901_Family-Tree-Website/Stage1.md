# Этап 1: Базовая структура и аутентификация

## Название этапа и его краткое описание

Разработка базовой структуры приложения и системы аутентификации пользователей. Создание основного каркаса веб-сайта с возможностью регистрации, входа и выхода пользователей.

## Цель этапа

Создать основу веб-приложения с рабочей системой аутентификации пользователей, готовой для дальнейшей разработки функциональности генеалогического дерева.

## Функциональные требования

1. Создание структуры Laravel приложения
2. Настройка базы данных SQLite
3. Реализация системы регистрации пользователей
4. Реализация системы аутентификации (вход/выход)
5. Создание базовых страниц (Главная, Логин, Регистрация)
6. Настройка middleware для защиты маршрутов
7. Создание миграций для пользовательских таблиц

## Нефункциональные требования

1. Безопасность аутентификации пользователей
2. Валидация входных данных
3. Защита от CSRF-атак
4. Хеширование паролей
5. Четкое разделение между аутентифицированными и неаутентифицированными пользователями
6. Производительность страниц не более 200ms

## Сценарии использования

### Сценарий 1: Регистрация нового пользователя
1. Пользователь переходит на страницу регистрации
2. Вводит имя, email и пароль
3. Подтверждает пароль
4. Нажимает кнопку регистрации
5. Система проверяет данные и создает учетную запись
6. Пользователь автоматически входит в систему

### Сценарий 2: Вход существующего пользователя
1. Пользователь переходит на страницу входа
2. Вводит email и пароль
3. Нажимает кнопку входа
4. Система проверяет учетные данные
5. При успешной аутентификации перенаправляет на главную страницу

### Сценарий 3: Выход из системы
1. Аутентифицированный пользователь нажимает кнопку выхода
2. Система завершает сессию пользователя
3. Перенаправляет на страницу входа

## Критерии приемки

1. Приложение успешно запускается и отображает главную страницу
2. Страницы регистрации и входа доступны по соответствующим маршрутам
3. Новые пользователи могут успешно зарегистрироваться
4. Существующие пользователи могут войти в систему
5. Защищенные маршруты недоступны без аутентификации
6. После выхода пользователь не имеет доступа к защищенным маршрутам

## Риски

1. Утечка персональных данных при неправильной настройке безопасности
2. Сложности с интеграцией Laravel Breeze или Jetstream (если будут использоваться)

## Зависимости

1. Фреймворк Laravel
2. База данных SQLite с существующей структурой таблиц (backend/database/migration/structure.sql)
3. Composer для управления зависимостями
